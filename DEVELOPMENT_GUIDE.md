# 🚀 คู่มือสำหรับนักพัฒนาในทีม (Team Onboarding Guide)

คู่มือนี้จะอธิบายขั้นตอนการนำโค้ดไปรันในเครื่องของตัวเอง (Local Development) ไปจนถึงการ Deploy อัตโนมัติขึ้น Railway ครับ

---

## 💻 1. การดึงโค้ดและรันในเครื่องตัวเอง (Local Development)

### ขั้นตอนที่ 1: ดึงโค้ดจาก GitHub
รันคำสั่งนี้ใน Terminal หรือ Command Prompt ในเครื่องของคุณเพื่อดึงโค้ดลงมา:
```bash
git clone https://github.com/hanif13/yala.git
cd yala
```

### ขั้นตอนที่ 2: ติดตั้ง Dependencies
โปรเจกต์นี้ใช้ Node.js รันคำสั่งนี้เพื่อโหลดแพ็กเกจทั้งหมดที่จำเป็น:
```bash
npm install
```

### ขั้นตอนที่ 3: ตั้งค่ารหัสผ่าน (Environment Variables)
ระบบมีการใช้งาน LINE Bot ซึ่งต้องการ Token ในการเข้าถึง เราจะไม่เก็บ Token ไว้บน GitHub เพื่อความปลอดภัย
1. ให้คัดลอกไฟล์ `.env.example` แล้วเปลี่ยนชื่อไฟล์ใหม่เป็น **`.env.local`**
2. ติดต่อ Admin (Hanif) เพื่อขอ `LINE_CHANNEL_ACCESS_TOKEN` และ `LINE_CHANNEL_SECRET`
3. นำค่าที่ได้มาใส่ในไฟล์ `.env.local`

### ขั้นตอนที่ 4: รันเซิร์ฟเวอร์จำลอง (Dev Server)
```bash
npm run dev
```
🎉 ตอนนี้คุณสามารถเปิดดูเว็บได้ที่ `http://localhost:3000` และเริ่มแก้โค้ดได้เลย! ระบบจะรีเฟรชหน้าเว็บอัตโนมัติเมื่อเซฟโค้ด

*(หมายเหตุ: หากต้องการให้ LINE Bot ตอบกลับตอนทดสอบในเครื่องตัวเอง ต้องรัน `ngrok http 3000` แล้วเอา URL ไปใส่ใน LINE Developers Console ชั่วคราว)*

---

## ☁️ 2. การ Deploy ขึ้นเซิร์ฟเวอร์จริง (Railway CI/CD)

เราได้เชื่อมต่อ GitHub (Branch: `main`) เข้ากับ **Railway** ไว้เรียบร้อยแล้ว ดังนั้นการ Deploy จึงเป็นไปโดยอัตโนมัติ 100%

### วิธีการนำโค้ดขึ้นเซิร์ฟเวอร์
เมื่อคุณแก้ไขโค้ดเสร็จแล้ว และต้องการนำขึ้นเว็บไซต์จริง ให้ทำตาม 3 คำสั่งนี้:

1. **บันทึกไฟล์ที่มีการเปลี่ยนแปลงทั้งหมดเตรียมไว้:**
   ```bash
   git add .
   ```

2. **เขียนสรุปสั้นๆ ว่าแก้เนื้อหาอะไรไปบ้าง (Commit Message):**
   ```bash
   git commit -m "อัปเดต: เปลี่ยนสีปุ่มแจ้งโรค และแก้บั๊กแผนที่"
   ```

3. **ส่งโค้ดขึ้น GitHub:**
   ```bash
   git push origin main
   ```
   *(หรือแค่ `git push` ถ้าผูก origin ไว้แล้ว)*

### เกิดอะไรขึ้นหลังจากกด `git push` ?
- โค้ดของคุณจะถูกส่งไปที่ `https://github.com/hanif13/yala`
- **Railway** จะตรวจจับอัตโนมัติว่ามีโค้ดชุดใหม่เข้ามา
- Railway จะทำการ Build และ Deploy ให้เองทันที (ใช้เวลาประมาณ 1-2 นาที)
- เมื่อเสร็จสิ้น เว็บไซต์จริง (Production) จะเปลี่ยนเป็นเวอร์ชันล่าสุดของคุณอัตโนมัติ! ✨

> 💡 **Tip:** สามารถเข้าไปดูสถานะการ Build แบบสดๆ ได้ที่หน้า Dashboard ของ Railway ว่าโค้ดที่เพิ่ง push ไปผ่านหรือไม่
